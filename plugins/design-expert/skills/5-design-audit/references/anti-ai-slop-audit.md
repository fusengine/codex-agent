# Anti-AI-Slop Audit

## What is AI Slop?

AI slop refers to generic, cookie-cutter design patterns that signal "this was generated by AI with zero customization." These designs lack personality, brand identity, and intentional choices.

## Detection Flags

### Font Check

| Flag | Indicator | Severity |
|------|-----------|----------|
| Inter | Default AI font choice | Major |
| Roboto | Android default, no personality | Major |
| Arial | System font fallback | Major |
| Open Sans | Generic web font | Minor |
| Lato | Overused template font | Minor |
| System UI only | No custom font loaded | Major |

**How to detect:**
```bash
grep -ri "inter\|roboto\|arial\|open.sans\|lato" src/ --include="*.css" --include="*.tsx" --include="*.ts"
```

**Fix:** Replace with intentional font pair from design-system.md typography section.

### Color Check

| Flag | Indicator | Severity |
|------|-----------|----------|
| Purple-pink gradient | `from-purple-500 to-pink-500` | Major |
| Hard-coded hex values | `#3b82f6`, `#6366f1`, etc. | Major |
| Default Tailwind palette | Using blue-500 without customization | Major |
| Default shadcn colors | Purple/zinc default theme unchanged | Critical |
| No CSS variables | Colors not tokenized | Major |

**How to detect:**
```bash
# Purple-pink gradients
grep -rn "purple.*pink\|violet.*rose\|indigo.*purple" src/ --include="*.tsx"
# Hard-coded hex
grep -rn "#[0-9a-fA-F]\{6\}" src/ --include="*.tsx" --include="*.css"
# Default Tailwind colors
grep -rn "blue-500\|indigo-500\|violet-500" src/ --include="*.tsx"
```

**Fix:** Define custom OKLCH palette in design-system.md and use CSS variables exclusively.

### Depth Check

| Flag | Indicator | Severity |
|------|-----------|----------|
| Flat cards without shadow | No elevation system | Minor |
| No glassmorphism or texture | Zero visual depth | Minor |
| Default white backgrounds | No surface variation | Minor |
| No dark mode | Only light theme available | Major |

**Fix:** Define elevation scale and surface colors. Add glassmorphism or subtle gradients for depth.

### Animation Check

| Flag | Indicator | Severity |
|------|-----------|----------|
| No animations at all | Static UI | Major |
| No hover effects | No interactive feedback | Major |
| No entrance animations | Content appears instantly | Minor |
| No loading states | No skeleton or spinner | Major |

**Fix:** Apply motion personality from design-system.md. Add Framer Motion to all interactive elements.

### Generic Pattern Check

| Flag | Indicator | Severity |
|------|-----------|----------|
| Default shadcn theme | Unchanged zinc/purple defaults | Critical |
| Stock illustrations | Undraw/unDraw without customization | Minor |
| Generic copy | "Lorem ipsum" or "Welcome to our platform" | Minor |
| No custom components | Only vanilla shadcn components | Major |
| No brand personality | Could be any company's app | Critical |

## Scoring

| Score | Meaning |
|-------|---------|
| 0 flags | Unique, intentional design |
| 1-3 flags | Minor customization gaps |
| 4-6 flags | Significant AI slop indicators |
| 7+ flags | Heavy AI slop - needs full identity redesign |

## Prevention

The best way to avoid AI slop:
1. Run identity-system skill BEFORE any design work
2. Generate design-system.md with unique tokens
3. Pass design-system.md to Gemini Design in every prompt
4. Audit after every major feature addition
