# Onboarding Copy Patterns

## Welcome Flow

### First Screen
```
Heading:     "Welcome to [App], [Name]!"
Subheading:  "You're 3 steps away from [core outcome]."
CTA:         "Let's go →"
```

### Step Progress
```
"Step 1 of 3 — Set up your profile"
"Step 2 of 3 — Connect your tools"
"Step 3 of 3 — Invite your team"
"All done! Here's what to do next."
```

## Tooltips

```
✓ "Click here to create your first project"
✓ "This is your inbox — all notifications land here"
✓ "Pro tip: use ⌘K to open the command palette anywhere"
✗ "This feature allows you to manage your workflow items"
```

## Empty States (first-time)

| Feature | Empty State Copy |
|---------|-----------------|
| Projects | "No projects yet" + "Create your first project — it takes 30 seconds" |
| Contacts | "Your contact list is empty" + "Import from CSV or add one manually" |
| Dashboard | "Your dashboard is ready for data" + "Connect a data source to get started" |
| Messages | "No messages yet" + "Invite teammates to start collaborating" |
| Reports | "No reports generated" + "Run your first report to see insights here" |

## Progress Indicators

```
"Profile 60% complete — add a photo to finish"
"3 of 5 integrations connected"
"You've completed 2 of 4 setup steps"
```

## Completion Celebrations

```
Minimal:  "You're all set!"
Warm:     "Your workspace is ready. Here's what you can do now:"
Playful:  "Nice work! You're officially set up."
```

## Re-engagement (after inactivity)

```
Email subject: "You haven't tried [Feature] yet, [Name]"
Body:          "You set up [App] 7 days ago. Here's the one thing
                most teams do in their first week: [specific action]."
CTA:           "Try it now →"
```

## Rules
- Name the outcome in step labels, not just the step number
- Empty states need both a headline AND a CTA — never just text
- Progress copy should highlight completion, not remaining work
- Tooltips max 12 words — if more is needed, link to docs
- Celebrate milestones without being patronizing
